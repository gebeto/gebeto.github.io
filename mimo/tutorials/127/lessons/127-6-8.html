A string is an ordered collection of characters. So far, so good. If we want a variable to take only a single character, we need to declare it as such.

let myCharacter: StringCharacter = "a"
let myString = "a"

In the case of myCharacter, type inference would create a string variable, which is why we need to explicitly declare it as a Character-type variable.

Wouldn't that be kind of unnecessary? We don't need to declare string variables if we give them a value at the same time.
