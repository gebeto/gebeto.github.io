Optionals are great because they handle the possible absence of a value.

It's not a good idea not to use "" or -1 to indicate that there's no value; optionals provide a much safer way.

funcmin(numbers: [Int]) -> Int? {
  if numbers.isEmpty {
    returnnil
  }
  varmin = numbers[0]
  for i in1 ..< numbers.count {
    if numbers[i] < min {
      min = numbers[i]
    }
  }
  returnmin
}

Optionals are a little more advanced but, used properly, they make life a lot easier.
