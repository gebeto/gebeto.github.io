External style sheets are nothing but text files with CSS code and a .css extension. Like HTML files, we can use any text editor to create them.

p {
 color: blue;
 font-size: 14px;
}
<!doctype html>
<html>
 <head>
  <link rel="stylesheet" type="text/css" href="style.cssindex.html">
 </head>
 <body>
  <p>I need CSS treatment.</p>
  <p>Me too.</p>
 </body>
</html>
<html>
 <head><style>p {color: blue;font-size: 14px;}</style>
 </head>
 <body><p>I need CSS treatment.</p><p>Me too.</p></body>
</html>
]]>

Sweet! The &lt;link&gt; tag consists of a type attribute that's "text/css", a rel attribute that's "stylesheet" and an href attribute that points to the CSS file.

Shouldn't the href attribute rather point to the CSS file?
